Rolling test

Rolling test is for testing frictional contact behaviors. We simulated simple system of primitive ball and box shaped objects. The error is measured by comparing the simulation with the analytical solution.

The test focuses on:

  1. The quality of the friction cone (linear or elliptic)
  2. The accuracy of frictional contact simulation

Test scenario

The setup of the test is as follows:

  • static ground
  • 10 kg box on the ground
  • 25 number of 1 kg balls on the box
  • the friction coefficient
    • 0.8 for ball-box pair
    • 0.4 for box-ground pari
    • 150 N force applied to box in the xy direction
  • ERP = 0 and CFM = 0
  • Metric: velocity of objects

Tested solver list

  • Rai
    • Bisection method
  • Bullet
    • Sequential Impulse
  • ODE
    • Dantzig
    • LCP PGS
  • MuJoCo
    • PGS
    • CG
    • Newton
  • Dart
    • Dantzig
    • LCP PGS

Note that ODE Dantzig solver fails when ERP is set to zero: the box goes into the ground as this video.

Results

The accuracy of RaiSim is bound by the contact solver threshold. This hyperparameter was not tunned for this test.

The pushing force resulted in no motion in ODE and Dart. However as the force is applied to y direction, the objects move. This is due to the pyramid shaped friction cone.

The objects oscillate significantly in MuJoCo as the time step size increases. It leads to an inaccurate solution.
